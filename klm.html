Keystroke-Level Model (KLM)

This page explains the Keystroke-Level Model (KLM) and how it predicts the time an expert user takes to complete simple tasks.

Introduction

The Keystroke-Level Model (KLM) is a predictive model introduced by Card, Moran, and Newell. It is based on the understanding of the human motor system and is used to make detailed predictions about expert user performance (Dix et al., Human–Computer Interaction, Ch. 12).

KLM predicts execution time for routine, highly practiced tasks, typically short in duration.. Examples include using search and replace or changing a font. It does not model complex problem-solving tasks. Those tasks must first be decomposed into subtasks, as in GOMS.

The model divides activity into two phases:

  • Acquisition Phase: The user builds a mental representation of the task.
  • Execution Phase: The user performs physical actions using the system.

KLM predicts performance only during the execution phase. The user is assumed to be expert and error-free.

KLM Operators

The execution phase is decomposed into physical, mental, and system operators (Dix et al., Ch. 12, p. 436):

  • K: Keystroke (pressing a key)
  • B: Pressing a mouse button
  • P: Pointing to a target
  • H: Homing (moving hand between keyboard and mouse)
  • D: Drawing using the mouse
  • M: Mental preparation
  • R: System response time

Total execution time is calculated as:

Texecute = TK + TB + TP + TH + TD + TM + TR

Operator times are derived from empirical studies. For example:

Typical Operator Times

Typical KLM Operator Times (Dix et al., Ch. 12)
Operator Description Typical Time (seconds)
K Keystroke (good typist) 0.12
B Mouse button press 0.10–0.20
P Pointing (average) 1.10
H Homing 0.40
M Mental preparation 1.35

Worked Exercise: Opening an Application

Three methods were analyzed: double-clicking, using an accelerator key, and using a menu. Results varied depending on input device.

The results show that device position and homing time significantly affect performance predictions.

Common HTML Errors

Test your understanding of semantic HTML, accessibility, and best practices with these simple, unique mistakes. Each question shows the correct answer and a short explanation.

  • Q1: What is wrong here? <header>My Page<footer>Footer</header>
    Answer: The footer is incorrectly placed inside header. Correct: <header>My Page</header><footer>Footer</footer> Explanation: Each semantic section should be separate to maintain document structure.
  • Q2: Identify the issue: <ul><li>Apple</li><li>Banana</li><p>Orange</p></ul>
    Answer: The p tag is inside ul, which is invalid. Correct: <ul><li>Apple</li><li>Banana</li><li>Orange</li></ul> Explanation: Only li elements are allowed inside ul or ol.
  • Q3: What’s missing here? <img src="logo.png">
    Answer: Missing alt attribute. Correct: <img src="logo.png" alt="Website Logo"> Explanation: Alt text is important for accessibility and when the image cannot load.
  • Q4: Why is this heading wrong? <h2>Section Title</h4>
    Answer: Closing tag does not match opening tag. Correct: <h2>Section Title</h2> Explanation: Matching heading tags preserves hierarchy and accessibility.
  • Q5: Identify the mistake: <a href="#"><li>Home</li></a>
    Answer: li cannot wrap a. Correct: <ul><li><a href="#">Home</a></li></ul> Explanation: Links should be inside list items, not the other way around.
  • Q6: What’s wrong with this form? <input type="text" placeholder="Name">
    Answer: Missing label for accessibility. Correct: <label for="name">Name</label><input type="text" id="name" name="name" placeholder="Name"> Explanation: Labels help screen readers identify form fields.
  • Q7: Find the nesting error: <div><section>Content</div></section>
    Answer: Tags are incorrectly nested. Correct: <div><section>Content</section></div> Explanation: HTML must have properly nested elements for valid structure.
  • Q8: What’s wrong with this table? <table><tr><td>Name</td><td>Score</td></tr></table>
    Answer: Missing thead and scope. Correct: <table><caption>Player Scores</caption><thead><tr><th scope="col">Name</th><th scope="col">Score</th></tr></thead><tbody><tr><td>Alice</td><td>10</td></tr></tbody></table> Explanation: Headers with scope improve accessibility and readability.
  • Q9: Identify the error: <p>Hello<p><span>World</span>
    Answer: First p tag not closed. Correct: <p>Hello</p><span>World</span> Explanation: Paragraphs must be closed before starting other elements.
  • Q10: Why is this navigation wrong? <nav><ul><li>Home</li><li>About</li></ul></nav>
    Answer: Links are missing. Correct: <nav aria-label="Main navigation"><ul><li><a href="index.html">Home</a></li><li><a href="about.html">About</a></li></ul></nav> Explanation: Navigation requires a tags for clickable links.
